Why no backend, here

The original PRD specifies a FastAPI backend with PostgreSQL for production. This build implements the same modules entirely client-side instead — which actually satisfies the PRD's strictest security rule (never store user passwords or encryption keys) by construction, since nothing is ever sent to a server. Phase 2+ items from the roadmap — accounts, cloud sync, sharing links — would be the trigger to introduce that backend.
